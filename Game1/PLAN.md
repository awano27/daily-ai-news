# PLAN.md

## 製品コンセプト

子供が iPhone で楽しく遊びながら、基礎的な計算力と数の感覚を身につける学習ゲームを作る。ゲームは短い時間で1プレイでき、正解するとポイントが増え、達成感を得られる構成にする。

このプロジェクトでは、一度に完成版を作らず、Phase 1 から順番に小さく動くものを作る。各 Phase では、その段階で必要な機能だけを実装し、次の Phase に進む前に完了基準を確認する。

## ターゲットユーザー

- 主な対象: 小学生低学年から中学年の子供
- 利用環境: iPhone のブラウザまたはホーム画面から起動できる Web アプリ
- 利用シーン: 家庭学習、すきま時間、親が近くで見守る学習時間
- 補助ユーザー: 保護者、先生、開発者

## 学習目標

- 足し算、引き算、かけ算、わり算の基礎を反復練習する
- 数字、演算記号、答えの関係を直感的に理解する
- 正確に計算する力を伸ばす
- 短い問題を連続して解く集中力を育てる
- 間違えた問題を責めず、もう一度挑戦できる体験にする

## 主要要件

- 子供向けの日本語表現を使う
- オリジナルの世界観、キャラクター、用語を使う
- 既存キャラクター、既存作品、商標、著作物に依存しない
- iPhone 画面で快適に使える UI にする
- safe-area を考慮し、ノッチ、Dynamic Island、ホームインジケータに UI が隠れないようにする
- 数学の問題、正解判定、ポイント計算は厳格に扱う
- 計算ロジックは UI 表示だけに頼らず、テスト可能な関数として分ける
- 一度に全部の機能を作らず、Phase ごとに実装する
- Phase 1 では最小限のゲームループを完成させる
- 実装後の報告では、完了内容、未完了内容、確認方法、次の作業を明確にする

## コアゲームループ

1. 問題が1問表示される
2. 子供が答えを選ぶ、または入力する
3. 正解または不正解をすぐに表示する
4. 正解ならポイントを加算する
5. 次の問題へ進む
6. 規定問題数または制限時間が終わったら結果画面を表示する
7. もう一度遊ぶか、次の難易度に進む

## Phase 1: 最小プレイ可能版

### 目的

ゲームとして成立する最小限の体験を作る。対象は iPhone での基本操作と、正確な問題生成、正解判定、ポイント加算に絞る。

### 実装範囲

- スタート画面
- 1種類の計算モード
- 問題表示
- 回答 UI
- 正解判定
- ポイント表示
- 結果画面
- もう一度遊ぶボタン
- iPhone 向けの基本レイアウト
- safe-area 対応
- 数学ロジックとポイント計算の基本テスト

### Phase 1 完了基準

- iPhone 幅の画面で、スタートから結果画面まで操作できる
- 10問など固定回数の問題を最後まで解ける
- 問題文、選択肢、答えが数学的に正しい
- 正解時だけポイントが増える
- 不正解時に誤ってポイントが増えない
- 合計ポイントが画面表示と内部状態で一致する
- 画面下部のボタンや重要表示がホームインジケータに隠れない
- 画面上部の見出しやスコアがノッチ、Dynamic Island に隠れない
- 子供向けの日本語がやさしく、命令口調や責める表現になっていない
- 既存キャラクター、既存作品名、商標を使っていない
- 開発者が実行方法と確認結果を報告できる

## Phase 2: 学習体験の拡張

### 目的

複数の問題タイプと難易度を追加し、子供の習熟に合わせて遊べるようにする。

### 実装範囲

- 足し算、引き算、かけ算、わり算の追加
- 難易度設定
- 出題範囲の調整
- 間違えた問題の再挑戦
- 連続正解ボーナス
- 結果画面の詳細化

## Phase 3: 継続利用の仕組み

### 目的

子供が繰り返し遊びたくなる要素を追加する。ただし学習を妨げる過度な演出や複雑な報酬設計は避ける。

### 実装範囲

- ステージまたはレベル進行
- 日ごとの記録
- 達成バッジ
- やさしい励ましメッセージ
- 保護者が成果を見やすい簡易記録

## Phase 4: 品質向上と公開準備

### 目的

公開できる品質に整え、保守しやすい構成にする。

### 実装範囲

- レスポンシブ UI の仕上げ
- アクセシビリティ確認
- パフォーマンス改善
- テスト追加
- データ保存の整理
- 公開手順の整備
- README や操作説明の整備

## リスクと対策

- リスク: 数学ロジックに誤りが入る
  - 対策: 問題生成、正解判定、ポイント計算を UI から分離し、テストを追加する
- リスク: iPhone の safe-area で操作ボタンが隠れる
  - 対策: `env(safe-area-inset-*)` を使い、実機相当の幅で確認する
- リスク: 子供向け表現が難しすぎる、または責める表現になる
  - 対策: 短く、ひらがな多めで、失敗時も再挑戦を促す文にする
- リスク: 既存 IP に似た表現になる
  - 対策: キャラクター、名前、世界観、画像、音声をすべてオリジナルにする
- リスク: Phase 範囲が広がりすぎる
  - 対策: Phase 1 の完了基準を満たすまで、ランキング、保存、複雑な演出を追加しない
